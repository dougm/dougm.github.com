Enum - HbrBaseGroupReplicationState(hbr.replica.BaseGroup.ReplicationState)


Enum Description

Describe the set of states that a group might be in with respect to replications from its primary.

Note these are only the Group states visible to VMODL clients. Internally, or to LWD clients, a Group might be in more sub-states (e.g., being deleted, or commiting, etc). Also there are no states to distinguish the various end-states (cleanedup, committed, rebalanced, etc) because the VMODL object is gone by the time those states are completed, so there is no place to report such states.

Because the Property Collector introduces latency between state updates and propagating those updates to clients, a short-lived property update may not be seen by the Property Collector.

State transitions at replica side may lag changes made at the primary site.

Regarding the 'fullsync' state:

  • for replicated VMs running on MN hosts: the HBR server cannot detect when a group "fullsync" is complete (the primary just stops sending "fullsync" updates), so that state will not change until an "lwd" is started, or the primary disconnects (and the state becomes "passive").
  • for replicated VMs running on an MN.next (or later) host: the HBR server will be notified when a full sync completes and the state will be switched to "opened"

Some "group" states are an imprecise amalgamation of per-disk states. For example, if during an "lwd" it happens that one of the disks in the group must go into "fullsync", the entire group state will change to "fullsync".

If an "lwd" or "fullsync" is aborted, the group state change may not get reflected here until the client disconnects (state becomes "passive") or another "lwd" or "fullsync" are initiated.

The HBR server cannot detect if a primary is "paused" so it cannot present that state here.

Enum Constants

NameDescription
fullsync fullsync - opened connection with ongoing full-sync. This state applies to all group types.
lwd lwd - opened connection with ongoing light-weight delta. This state applies to all group types.
opened opened - an open connection from a primary, but no active fullsync or lwd. This state applies only to a ReplicationGroup.
passive passive - configured but no active or open connections. This state applies to all group types.
paused paused - replication is paused. This applies to group type derived from CascadedGroup: CloudStorageGroup.
