Data Object - HbrInstanceQuerySpec(hbr.replica.InstanceQuerySpec)

Parameter to
GetInstanceCount, GetInstances
Extends
DynamicData

Data Object Description

Specification to query for instances by their timestamp (@link GroupInstanceStats#timestamp}. It specifies a time window for searching for instances.

To query for instances with timestamp equal to or earlier than a given time point, set endTime to the time point.

To query for instances with timestamp equal to or later than a given time point, set startTime to the time point.

To query for the latest N quiesced instances, set maxInstances to N and set quiesced to true.

To query for the quiesced instances within a specific time window, set {endTime, startTime and quiesced.

Properties

Name Type Description
endTime*xsd:string

Query for instances with timestamp no later than endTime. Any timestamp qualifies if not specified.

The format is "YYYY-MM-DD HH:MM:SS UTC". HH is 0-23.

maxInstances*xsd:long

Maximum number of instances returned by the query. This is to alleviate performance impact on the system if there are a large number of instances satisfied the query. The same as the system default which is XXX if not specified. Negative value means no limit. The system default can be updated via hbrsrv config option XXX. The server will return the most recent maxInstances if the number of qualified instances is larger than maxInstances. XXX option to return the last maxInstances? This is ignored by GetInstanceCount.
quiesced*xsd:boolean

Query for quiesced instances.

If not specified, any instances regardless quiesced or not will be returned.

If it is true, return applicatin-quiesced or file-quiesced instances.

Otherwise, return non-quiesced instances.

startTime*xsd:string

Query for instances with timestamp no earlier than startTime. Any timestamp qualifies if not specified.

The format is "YYYY-MM-DD HH:MM:SS UTC". HH is 0-23.
