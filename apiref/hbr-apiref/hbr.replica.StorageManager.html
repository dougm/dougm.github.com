Managed Object Description

Interface to control hosts and datastores for the the host-based replication secondary site.

Configuration

Configuring an HBR server involves informing the server of available hosts, and for those hosts which datastores should be accessed. The set of datastores, hosts, and preferred hosts can be updated at any time both through configuration API or connectivity events in real time.

Returns all of the hosts configured on this hbrsrv.

While this could just be a method since the caller initiates host enablement, allowing this to be a property means that caller can use the property collector to monitor hosts as a group through the more advanced features of the property collector.

expectedDatastoresList*ManagedObjectReference[]
to a HbrDatastore[]

Returns the the list of datastores that the HBR server will access in support of its configured replications. This list will contain all of the datastores that have been configured for some group (via a HbrGroupSpec or image (via HbrReplicationGroupCreateTestBubbleImage_Task or HbrReplicationGroupCreateFailoverImage_Task).

While this could just be a method because the caller supplies the datastores and knows when to refresh the list, allowing this to be a property means that the caller can monitor datastores as a group through the more advanced features of the property collector.

HbrStorageManagerEnableHost(enableHost)

Enable the HBR server to access the given host and the datastores attached to that host. The given hostAuth credentials will let us in.

The HBR server will discover the set of available datastores directly from the host.

The call to EnableHost is idempotent. Adding the same host multiple times should end up modifying any host attributes that have changed.

In all cases, the call causes the server to re-establish the connection to the host. This is useful e.g. to force a quicker reconnect after the hbrsrv thumbprint has been pushed into the host.
