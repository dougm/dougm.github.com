Data Object Description

A GroupInstanceData represents a specific point-in-time replica of a group.

Properties

Name Type Description
instanceStatsHbrGroupInstanceStats

Get the UTC timestamp, transfer time, and transfer byte counts of this instance.
keyxsd:string

An identifier for this instance. This can be used as a tag for the system to identify an instance.

This identifier is provided by the primary side when an instance is created for a protected VM. For cascaded replication, the identifier is propagated and applied to the instances of the cascaded groups when a source instance is replicated to these groups.

Note that it is named as "key" per the requirement of property update. A data object must have a property named "key" for partial updates via _RecordAdd or _RecordRemove (i.e. property updates for latestInstances).

parentId*xsd:string

Identifier of the parent instance (key) in the same snapshot series.

A parent instance is the one which was created right before the current instance in the same snapshot series.

quiescedTypeHbrVirtualMachineInstanceQuiescedType

Type of quiescing
rpoViolationxsd:int

Return the RPO violation for this group instance (in minutes since the deadline elapsed). Returns 0 if the group instance is within its RPO deadline. Returns -1 if the group instance is not completed.

We update this property when the group instance is completed.

sequenceNumberxsd:long

A sequence number identifying this instance. Each instance of a group has a sequence number strictly larger than that of all previous instances.
snapshotTimestampxsd:string

Timestamp when the instance was taken for the protected VM.

For cascaded replication, the instances recording the same VM state have the same snapshotTimestamp. This timestamp is generated by the hbrsrv which creates the source instance. It is propagated and applied to the instances of the cascaded groups when a source instance is replicated to these groups.

This property is immutable and it will not be adjusted based on the local hbrsrv clock.

The format is "YYYY-MM-DD HH:MM:SS UTC". HH is 0-23.
