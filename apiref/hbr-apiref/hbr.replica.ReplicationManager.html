Managed Object Description

Interface to control the host-based replication secondary site configuration. Also interface to do bulk stats query for hosts, groups and server.

Configuration

Configuring an HBR server involves informing the server of the storage configuration. See the HbrStorageManager for details.

Replication Groups

Each replication group added to the server enables the VMs in that group to use this server as a replication target (e.g., HbrReplicationGroup).

A Replication Group contains one or more VMs (just one in version 1). Each VM has a set of virtual disks associated with it. The disks within a group are replicated in a consistent manner (crash-consistent, file-consistent, or application-consistent) as a set.

Groups are defined and configured with specs that define the elements (see HbrGroupSpec, HbrVirtualMachineSpec, HbrDiskSpec).

Once a group is registered, it can be queried about its basic configuration (see HbrReplicationGroup, HbrVirtualMachine, HbrDisk).

Each group has zero or more consistent instances of the group. Each instance represents a single point-in-time replica of the source group. See HbrGroupInstanceData.

Each instance can be used to create bootable images of the group. See HbrGroupImage, HbrVirtualMachineImage, HbrDiskImage, HbrConfigFileImage.

Note that the replicated virtual machines are not in a completely bootable form. At a minimum the VM configuration file will need to be patched up to reference the replicated disks (the replica disk paths need not match the original paths on the replica side).

Missing Features

  • Authentication and authorization of the VMODL client
  • Pushing asynchronous events to clients
  • Exception signatures, exception types
  • GroupConfigurationCheck -- check that Spec config is 'valid' for a strong definition of valid.

SetPrimaryHosts_Task(setPrimaryHosts)

Specify a full set of primary hosts that belong to a trusted site.

If given host is not registered with the server then it will be added, otherwise it will be updated. A host that is already registered with the server but not present in the spec array will be removed.

LWD connections already established on behalf of updated or removed host will be closed.
