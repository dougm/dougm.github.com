Managed Object Description

The GroupImage represents a valid, point in time replica for the associated Group. The image could refer to other point in time images which are registered as snapshot of the virtual machine. The HBR configuration of a GroupImage is not directly mutable.

There are two kinds of images: test bubble and failover. A test-bubble image (see HbrReplicationGroupCreateTestBubbleImage_Task) is expected to be reverted after any testing is completed. A failover image (see HbrReplicationGroupCreateFailoverImage_Task will either be reverted or "committed" (HbrReplicationGroupCommitToImage_Task).

A revert of an image (either kind) will generally clean up any files created to make that image. In the case of a test-bubble image, the entire containing directory of the image will be removed, along with all content (and hopefully any of the VM's auxiliary files). For a fail-over image, the image directory is not private to the image, and so only the files explicitly created by the hbrsrv will be cleaned up.

Note that any errors cleaning up an image (e.g., due to an unreachable datastore) may leave those image files behind. For a reverted fail-over image, this should be okay (the next failover will simply overwrite the files).
