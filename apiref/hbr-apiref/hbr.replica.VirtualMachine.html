Managed Object Description

Represents an in-progress, virtual machine replication. Each virtual machine is part of a HbrReplicationGroup.

HbrVirtualMachineMove_Task(move)

Move the secondary-side storage for this virtual machine.

This method can be used to move any or all secondary-side disks to a new location, as well as the VM's directory containing non-disk replication state.

The function returns a HbrReplicaTask which can be used to monitor the progress of the operation. The extraInfo field for the task will be populated with the given new HbrVirtualMachineSpec.

The call is idempotent - if the connection is lost as the call is made, the call can be retried; if the move operation started already, the reference to the existing HbrReplicaTask is returned. If the operation completed already, the call will be a no-op and the task will finish immediately.

If the server dies during the operation, the operation resumes after restart, and the same task object reference is still valid.

Note that the HbrReplicaTask object exists for some time after the operation completes, but it is eventually cleaned up.

If the operation is canceled or encounters an error, some subset of the disks (and/or the non-disk replication data) may have successfully moved to the new location, while others may be restored to their old location. Any individual disk will either be in the old or new location, but the set of disks may not match either the old or the new spec. In any case, retrying the operation with the same new spec is always possible, and if that completes successfully, the new spec will be in effect.

UnconfigureDisk(unconfigureDisk)

Unconfigure a disk from this virtual machine. After this call completes, data for this disk will not be accepted from the primary, and no consistent instances will be created as long as the primary still replicates the corresponding disk.

Existing group instances or images still using the disk are NOT removed. The disk hierarchy remains under HbrServer's control until a successful call to HbrVirtualMachineRemoveDisk_Task.

If the disk was already unconfigured, the function does nothing (no exceptions).
