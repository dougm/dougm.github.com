Data Object - HbrRetentionPolicyTier(hbr.replica.RetentionPolicy.Tier)

Property of
HbrRetentionPolicy
Extends

Data Object Description

A "tier" specifies that the server will try to retain instances that span numSlots "slots" of granularityMinutes minutes.

For example, to retain one hourly instance for the last 3 hours, set granularityMinutes=60, numSlots=3. Note that the latest instance will always be retained. This means a total of 4 instances will be retained in this example.

granularityMinutes can set to 0 to retain the 'numSlots + 1' latest instances.

The slots are aligned to "round" dates/times; for example:

  • slots of granularity 5 minutes would be delimited at times of the form xx:x5
  • slots of granularity 1440 (one day) would be delimited at 12:00am

Properties

Name Type Description
granularityMinutesxsd:int

Slot granularity, in minutes. It can be 0 or a positive value. A value of 0 can be used to specify tier for latest instances.
numSlotsxsd:int

Number of slots. This value must be at least 1. Note that when the number of slots is 1, the only instance that is retained may be too new. For example, if retaining 1 instance each day with RPO of 60 minutes, if instances gets created at 12:45am and 1:45am, the 1:45am instance will get retained as the latest instance and 12:45am instance will get retained as the daily instance.
